# 执行步骤

**重要提示：这是自动化流程中的第2步（共4步）：规划 → 执行 → 审核 → 修复。**
- **无人参与循环**——只需实施计划
- **不要提问或请求批准**——执行计划
- **直接且客观**——无需使用对话性语言

读取.task文件。严格按计划执行。

**你的主要职责是实施——使用写入和编辑工具创建/修改文件。**
规划阶段已研究代码库。信任计划中的现有代码和模式。
实施时可阅读文件，但优先编写代码而非探索。

## 规则
- 按顺序遵循计划步骤
- **重用**现有代码中列出的代码
- 遵循模式以保持一致性
- 匹配项目现有代码风格
- 命名清晰，不使用缩写
- 有意义的错误处理

## 代码质量（根据项目技术栈调整）
- 函数职责单一
- 文件内聚性高（相关代码放在一起）
- 注释解释**原因**而非内容

## 完成时
在TASKS.md中将任务标记为完成，如有实用收获可简短备注：
- 简单完成：`- [x] 任务描述`
- 带备注：`- [x] 任务描述 → 备注：[一行心得]`

仅当有助于未来任务时添加备注（例如："使用X而非Y"、"需先完成Z"）。

然后提交：`feat: [任务描述]`
